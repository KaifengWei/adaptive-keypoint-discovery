# 秧苗自适应关键点人工复核协作体系

本目录保存人工复核体系的长期说明与可复现打包脚本。人工复核是论文验证门槛，不是模型组件，不参与自动教师或关键点检测器训练。

## 两级复核、四个平台

1. **地上部有效域复核**：检查地上部是否完整、颖果根须是否排除、shoot侧基部过渡区是否合理。
2. **路线B关键点条件器官路径复核**：检查路径语义、漏叶、错连和基部选择。
3. **方案C独立路径复核**：使用完全相同的字段复核新教师与新解码器，禁止先看方法对照。
4. **二维拆分方法对照**：比较旧/新教师和全局/局部短枝规则四种组合，只用于定位改进来源。

## 推荐协作方式

- 每名复核者独立打开资料包中的HTML，不提前查看其他人的完成示例；
- 复核方案C时先完成平台三，再打开平台四；
- 完成全部图像后点击“导出CSV”；
- 文件命名为`平台类型_复核者_日期.csv`；
- 汇总时保留各人的原始结果，不覆盖，不互相抄录；
- 先计算一致率与Cohen's kappa，再对分歧样本共同裁决。

## 生成可移交资料包

```powershell
python .\build_review_collaboration_package.py `
  --output D:\kp\人工复核协作资料包_20260723 `
  --zip D:\kp\人工复核协作资料包_20260723.zip
```

脚本会复制两个离线HTML、全部依赖图片、空白CSV、已完成示例、阶段报告，并生成SHA-256清单。压缩包不包含checkpoint、原始大数据或test模型结果。
