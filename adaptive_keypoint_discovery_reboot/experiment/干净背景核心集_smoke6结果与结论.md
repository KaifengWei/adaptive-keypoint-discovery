# 干净背景核心集 smoke6 复测结果与结论

## 1. 正确的结果导向

本研究的关键点不是预先命名的器官类别，也不要求必须是少量、离散、不可替代的解剖事件。它们是算法根据当前秧苗形态自动产生的二维支撑点集合，最终服务于：

1. 植株骨架和各分支曲线的重建；
2. 样条拟合与沿曲线抽样；
3. 株高、路径长度、宽度、叶倾角、弯曲度等表型计算；
4. 由多个表型形成秧苗健壮度和农艺指标评价。

因此，点集较密本身不是失败。只有当增加的点不能改善曲线或表型精度，或导致噪声、错误连线和不稳定时，才应视为冗余。此前使用“是否形成少量不可替代结构事件”作为停止条件，与本研究目标不一致，现已撤销。

## 2. 数据准备结果

从 `D:\kp\数据集图片` 共盘点 2239 张图像，其中包含400张原始图、1200张增强图和639张扫描采集图。增强图不参与独立选样。最终建立20张干净背景核心集，分为6张 smoke 和14张后续集。20张之间无完全重复，在 `dHash <= 8` 条件下也未发现近重复对。本阶段固定使用该核心集，不再更换数据来源。

## 4. G1′已经解决和尚未解决的问题

G1′已经初步解决：

- 在单张图上自动产生数量可变的候选点；
- 候选点主要落在秧苗茎叶结构上；
- 在轻微几何和光度扰动下保持较高重复性；
- 点集对植株骨架具有较高覆盖率。

G1′尚未解决：

- 当前点坐标只是无序集合，还没有按主轴、叶片或分支进行路径分组；
- 尚未确定每条路径中点的先后顺序；
- 尚未验证这些点拟合出的样条曲线是否准确；
- 尚未验证由曲线、掩膜和拓扑计算出的表型是否接近参考测量值；
- 当前20点是安全容量上限，`min_distance`仍是尺度相关规则，后续需要检查它们是否限制复杂植株的自适应点数。

## 5. 修订后的阶段结论

本轮结论为：**数据质量确实影响方法表现；在干净背景受控数据上，G1′已经通过“自适应表型支撑点候选”阶段，应进入表型效用验证。**

剩余14张不再因为“点太密”而被否定。它们暂时保留，是为了先完成统一的点坐标导出、骨架图构建、分支排序和表型计算接口，然后一次性验证完整链路，避免只用点数和覆盖率评价最终效果。

## 6. 下一阶段：点集到表型的验证桥梁

下一阶段采用以下闭环：

```text
G1′自适应候选点
  → 在前景骨架上投影
  → 构建骨架图并识别端点、分叉点与路径
  → 将候选点分配到各条路径并按测地距离排序
  → 对每条路径拟合B样条或平滑样条
  → 结合掩膜计算长度、宽度、角度、曲率、面积和拓扑数量
  → 与骨架像素参考和人工表型测量进行比较
```

评价分为两层：

| 层级 | 回答的问题 | 主要指标 |
|---|---|---|
| 点集可靠性 | 点是否稳定落在有效植株结构上 | 前景命中率、重复性F1、定位误差、点数扰动 |
| 表型效用 | 这些点是否足以得到准确表型 | 曲线Chamfer距离、骨架覆盖、长度/角度/曲率误差、表型MAE、MAPE、R²与一致性 |

最终点数不由“越少越好”决定，而由表型误差与点集复杂度的关系决定：当继续增加点已不能显著降低曲线重建或表型误差时，当前点数才是该植株和该任务下的合理数量。
