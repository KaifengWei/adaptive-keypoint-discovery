# 远程 GPU 正式训练执行说明

## 1. 先确认你真的在远程终端

在 VS Code 左下角确认显示 `SSH: ...`，然后“终端 → 新建终端”：

```bash
hostname
nvidia-smi
cd /你的远程路径/adaptive_keypoint_discovery_reboot
/media/neaucs2/evs/envs/adaptive_kp/bin/python remote_gpu_check.py
```

最后一条必须显示 `torch.cuda.is_available() == true` 才能训练。`nvidia-smi` 只证明驱动看得到显卡，不证明当前 Python 装的是 CUDA 版 PyTorch。

本服务器的登录 shell 还继承了系统 CUDA 11.4/cuDNN 9.4 路径，而项目独立环境中的 PyTorch wheel 使用 CUDA 12.8 和随包 cuDNN。项目入口 `run_remote_core.sh` 会在当前子进程中清除 `LD_LIBRARY_PATH`，避免加载旧版 cuDNN；不会改动服务器的全局 CUDA 配置。若手工单独运行训练命令，应先执行：

```bash
unset LD_LIBRARY_PATH
```

## 2. 建环境

当前服务器已经建立项目专用环境 `/media/neaucs2/evs/envs/adaptive_kp`，不修改共享的 `kf` 环境。新服务器或需要重建时执行：

```bash
cd /home/neaucs2/kp/adaptive-keypoint-discovery/adaptive_keypoint_discovery_reboot/experiment
bash setup_remote_env.sh
source ~/anaconda3/etc/profile.d/conda.sh
conda activate /media/neaucs2/evs/envs/adaptive_kp
unset LD_LIBRARY_PATH
```

脚本固定 Python 3.12，使用官方 Conda 源安装科学计算依赖，并从服务器本地 wheelhouse 安装 `torch 2.9.1+cu128` 与 `torchvision 0.24.1+cu128`。它不会写入全局 `.condarc`。`requirements_remote.txt` 是依赖范围说明；PyTorch 仍单独安装，防止误装 CPU 包。

## 3. 检查必要文件

```bash
test -f third_party/checkpoints/dinov2_vits14_reg4_pretrain.pth

python -c "from pathlib import Path; print(len(list(Path('data_stage_clean_v3/images').rglob('*.png'))))"
```

最后一条应为 98。配置和伪标签都使用相对路径，服务器不需要存在 `D:` 盘。

## 4. 生成核心自动伪标签

```bash
python generate_g1prime_pseudolabels.py \
  --dataset data_stage_clean_v3 \
  --output pseudo_labels_g1prime_v3 \
  --splits train auxiliary \
  --device cuda \
  --full-transforms
```

输出：

- `pseudo_labels.jsonl`：每图数量可变的坐标、置信度、重复出现比例；
- `pseudo_label_audit.csv`：每图接受/拒绝点数和是否训练可用；
- `overlays/`：接受点与拒绝点可视化；
- `summary.json`：全量可用率和点数统计。

如果 `training_usable_images` 明显低于总图数的 80%，先停止，不进入训练。

## 5. 核心模型先干跑，再正式训练

```bash
python train_adaptive_point_detector.py \
  --config configs/train_core_dinov2.json \
  --dry-run

python train_adaptive_point_detector.py \
  --config configs/train_core_dinov2.json
```

`dry_run.json` 应显示输入、目标和输出形状一致。正式输出：

- `best.pt`：最佳 checkpoint；
- `history.csv`：每轮训练损失、验证损失和伪标签 F1；
- `training_summary.json`：训练结束状态；
- `resolved_config.json`：服务器上实际使用的完整路径和参数。

RTX 3090 建议先用配置中的 batch 4、梯度累积 2。若显存不足，把 batch 改为 2；不要先降低图像尺寸，因为 518 与 DINOv2 14 像素 patch 网格严格匹配。

## 6. 锁定 val/test 评估

```bash
python evaluate_adaptive_point_detector.py \
  --checkpoint training_outputs/core_dinov2/best.pt \
  --dataset data_stage_clean_v3 \
  --output evaluation_outputs/core_dinov2 \
  --device cuda \
  --splits val test \
  --full-transforms
```

输出点坐标、跨变换 F1、定位误差、前景命中、路径、样条、表型 CSV 和叠加图。`phenotype_accuracy_status` 在人工参考未填前仍会明确写 `pending`。

## 7. 四项核心消融

先生成两套教师伪标签：

```bash
python generate_g1prime_pseudolabels.py \
  --dataset data_stage_clean_v3 --output pseudo_labels_geometry_v3 \
  --splits train auxiliary --device cuda --full-transforms \
  --teacher-variant geometry_only

python generate_g1prime_pseudolabels.py \
  --dataset data_stage_clean_v3 --output pseudo_labels_no_consistency_v3 \
  --splits train auxiliary --device cuda --full-transforms \
  --no-consistency-filter
```

训练配置：

| 配置 | 检验问题 |
|---|---|
| `train_core_dinov2.json` | 完整方案 |
| `train_ablation_geometry_teacher.json` | 去掉 DINOv2 教师证据后怎样 |
| `train_ablation_no_consistency.json` | 去掉跨变换筛选后怎样 |
| `train_ablation_fixed_k.json` | 强制固定 14 个点是否更差 |
| `train_ablation_unfreeze_last2.json` | 冻结骨干与微调后两层的差别 |

逐个运行 `train_adaptive_point_detector.py --config ...`，再用相同 val/test 评估。不要用 test 选择阈值；阈值只能由训练内部验证或 val 确定。

## 8. 五随机种子

核心方案稳定后再跑：

```bash
for seed in 20260717 20260718 20260719 20260720 20260721; do
  python train_adaptive_point_detector.py \
    --config configs/train_core_dinov2.json \
    --seed "$seed" \
    --output-dir "training_outputs/core_dinov2_seed_${seed}"
done
```

消融先各跑一个种子；只有核心结论稳定后再扩展五种子，避免在错误方案上消耗算力。
