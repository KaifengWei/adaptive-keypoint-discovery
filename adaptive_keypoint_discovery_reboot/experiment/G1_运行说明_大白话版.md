# G1 怎么运行、会得到什么

G1 不训练模型，也不需要你标关键点。它只把官方 DINOv2 冻结起来，观察模型特征里自然出现的候选点，在图像翻转、旋转、缩放、平移、亮度和对比度变化后还能不能回到原来的结构位置。

## 需要准备的文件

1. 已由 G0 生成的 `data_g0/pilot80_manifest.csv` 和 `data_g0/crops`。
2. 官方 DINOv2 代码仓库。
3. 官方 `dinov2_vits14_reg` 权重。联网设备可以由 `torch.hub` 自动下载；离线设备应提前下载代码和权重，并通过 `--local-repo`、`--weights` 指定。
4. 完整实验建议使用 CUDA。当前电脑只有 CPU，适合环境检查和 1～2 张烟雾测试，不适合一次跑完 80 张、多分辨率和全部变换。

## 四条命令

先只检查文件和环境：

```powershell
python .\g1_dinov2_feasibility.py --phase verify
```

下载好模型后，用 2 张图检查代码链路：

```powershell
python .\g1_dinov2_feasibility.py --phase smoke --sizes 518 --layers last4avg --limit 2
```

在 20 张校准图上确定候选峰值阈值：

```powershell
python .\g1_dinov2_feasibility.py --phase calibrate
```

冻结阈值后，在剩余 60 张先导图上作出 G1 门禁结论：

```powershell
python .\g1_dinov2_feasibility.py --phase pilot
```

先导阶段不会重新挑参数。程序只运行 20 张校准集为三种候选方法分别选出的全局最优配置，并把选择写入 `pilot/selected_configurations.json`。

离线 CUDA 设备示例：

```powershell
python .\g1_dinov2_feasibility.py --phase calibrate `
  --local-repo "D:\bundle\dinov2" `
  --weights "D:\bundle\dinov2_vits14_reg4_pretrain.pth" `
  --device cuda
```

## 输出如何看

- `calibration/frozen_thresholds.json`：20 张图确定、以后不再改的阈值。
- `pilot/summary_metrics.csv`：各分辨率、特征层和候选方法的汇总结果。
- `pilot/g1_gate_decision.json`：自动门禁是通过还是停止/修改。
- `pilot/visualizations`：候选点叠加图。人工只看这些点是否反复落在结构相关区域，不用给点命名。
- `per_transform_metrics.csv`：每一种变换的匹配精度、召回率、F1 和归一化定位误差。

自动指标全部通过后，仍需两个人独立查看可视化。只有两位复核者都认为至少 70% 图像出现可重复、结构相关而非纯背景的候选，G1 才算完整通过。此处的“通过”只说明发现了稳定候选，不能直接宣称它们已经具有生物学关键点含义。
