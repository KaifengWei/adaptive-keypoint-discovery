# G1′—V3 阶段验证结论

## 1. 这轮实际验证了什么

本轮没有训练最终模型，也没有把自动候选点当成生物学真值。验证对象分为三层：

1. 数据是否足够干净，能否避免背景纹理制造假骨架；
2. G1′候选点能否连接、排序成茎叶路径，并输出样条表型；
3. G1′能否通过跨变换共识，自动形成可供模型学习的数量可变伪标签。

## 2. 数据结论

正式数据目录为 `data_stage_clean_v3`，共 98 张：

- 30 张带用户提供的时期元数据，拆为 train 19、val 5、test 6；
- 68 张时期代理或未知图，只进入 `auxiliary`，不参与时期结论；
- 1 张重复图被排除；
- 2 张在标准化后发生真叶断裂，被排除；
- 未读取任何关键点标签。

所有训练图统一为地上部目标。原图来源、处理方式、拆分、质量复核和近重复审计均保存在 manifest 中。

## 3. G1′路径结果

在 11 张锁定 val/test 图上：

- 候选点数中位数：17；
- 路径数中位数：2；
- 11/11 均生成了连通骨架图；
- 安全上限触发率：0；
- 三个时期均有覆盖；
- 已输出候选坐标、点类型、基部、路径、样条、像素长度、归一化长度、弦长、曲折度、总转角、平均曲率和分支角。

这些数字只证明计算链路连通，不证明路径都正确。视觉总览显示至少两类真实失败：

1. 横置、灰暗样本可能把叶尖错当基部，例如 `stagev3_0002`；
2. 个别叶片虽有候选点，但骨架路径没有完整覆盖，例如 `stagev3_0015`。

因此当前结论是：

- “G1′点候选 → 自动伪标签”可以继续；
- “G1′路径 → 表型精度已成立”暂不通过，必须完成 11 张人工路径复核和人工表型参考测量。

复核入口：

- `outputs_g1_prime_v3/phenotype_primary11/路径视觉复核总览.jpg`
- `outputs_g1_prime_v3/phenotype_primary11/路径视觉复核表_待填写.csv`

## 4. 自动伪标签结果

3 张 CPU 冒烟图使用 identity、水平翻转、旋转 10°、亮度 0.75 四个视图：

- 每图原始候选中，不稳定点被自动拒绝 2—7 个；
- 最终接受点数为 9、14、14；
- 3/3 图满足训练可用门槛；
- 没有人工关键点坐标参与。

这证明自动教师链路可以运行，但 3 张结果不能代替全量统计。全量 87 张 train+auxiliary 伪标签应在远程 GPU 上重新生成。

## 5. 可学习模型状态

已经建立：

- DINOv2 ViT-S/14 reg 冻结骨干；
- 约 24 万参数的密集热图头（CPU 冒烟配置）；
- 局部极大值 + 置信阈值决定点数，不设固定 K；
- 高斯伪标签、热图损失、计数约束、训练/验证拆分、checkpoint；
- 训练后跨变换重复性、前景命中、路径和表型输出入口；
- 几何教师、去一致性筛选、固定 K、解冻最后两层等消融配置。

CPU 已完成一次真实反向传播和一次训练后评估。单步随机头的独立图前景命中率约 0.018，属于预期失败，只证明评估代码没有掩盖未训练模型。

## 6. 当前门槛决策

| 门槛 | 状态 | 含义 |
|---|---|---|
| 数据背景纯净与拆分独立 | 通过 | 可进入训练 |
| G1′候选可重复伪标签入口 | 冒烟通过 | 远程生成全量后再统计 |
| 动态模型训练代码 | 通过 | 已完成 forward/backward/checkpoint/eval |
| 11 张路径语义正确 | 待人工复核 | 不得跳过 |
| 表型误差 | 未验证 | 缺人工测量参考与物理尺度 |

## 7. 下一步顺序

1. 远程检查 GPU 和 CUDA PyTorch；
2. 生成全量 G1′跨变换伪标签；
3. 先跑核心模型一个种子的 dry-run 和正式训练；
4. 锁定阈值后评估 val/test；
5. 填写 11 张路径复核表，先解决基部歧义和漏叶；
6. 核心链路通过后再跑四项消融和五随机种子；
7. 有人工表型参考后，才能报告最终表型误差。
