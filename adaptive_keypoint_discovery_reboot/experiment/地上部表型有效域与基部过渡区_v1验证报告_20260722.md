# 地上部表型有效域与基部过渡区 v1 验证报告（2026-07-22）

## 1. 本轮目标与边界

根据40张 V4 val 人工路径审计暴露出的系统性根须干扰，本轮实现路线B的第一阶段：在保留完整整株图的同时，自动提出只用于地上部关键点、器官路径和表型计算的 `phenotype_roi`，并在芽鞘节/茎基部附近提出 shoot侧 `basal_transition`。

本轮只读取既有 V4 `val=40` 图像、自动器官掩膜、已保存的旧模型学习点和人工路径审计表；没有运行模型，没有生成新教师，没有训练，没有修改任何数据样本或split，也没有读取 V4 test。

## 2. 方法定义

### 2.1 完整保存与分析有效域分离

完整标准化 RGB 继续保留颖果和根系，作为可追溯图像和器官定位上下文。`phenotype_roi` 只决定哪些像素允许产生地上部关键点和路径，不覆盖或删除原始数据。

### 2.2 不使用固定“图像上方”规则

程序先利用褐色颖果候选与 `seed_base_root` 掩膜定位其邻近的shoot边界，再由远端绿色shoot像素估计茎叶延伸方向。有效域保留该边界沿茎叶方向一侧的shoot，并加入窄的低饱和基部过渡带；颖果和大部分根系位于相反一侧而被排除。该方向随每株苗自动变化，适用于横放、竖放和旋转图像。

### 2.3 基部仍必须来自学习点

自动过渡区不是人工定义关键点，也不能直接作为最终基部坐标。候选地上部起算点仍必须来自已有学习型热图模型输出，并且满足：位于有效域容差内、靠近shoot侧过渡区。没有合格学习点时明确输出 `NO ELIGIBLE LEARNED BASAL ANCHOR`，不得回退到颖果或根须。

## 4. 初步解释

- 有效域在40张图上均保留至少90%的旧shoot掩膜，同时把seed/base/root重叠中位数降到不足1%，说明“完整留档、地上部分析分域”在像素层面可行。
- 旧模型只有37.2%的学习点落在新有效域，和人工审计中38/40条备注指出根系/根须干扰相符，说明旧教师与旧模型确实学习了大量本研究不需要的组织响应。
- 仅6张图存在靠近新过渡区的旧学习点，并不表示新有效域失败。旧模型从未接受“地上部有效域”约束，缺少shoot侧基部点正是需要重建教师和重新训练的原因。
- 当前不能直接使用新有效域重新解码路径，因为34张图没有合格旧学习起算点；也不能把自动掩膜保留率写成器官分割精度。

## 5. 人工复核页面说明

入口：`evaluation_outputs/phenotype_roi_basal_anchor_v1_val/地上部有效域复核.html`。

每张图分三栏：

1. 完整标准化图；
2. 绿色为拟保留的地上部有效域，黄色为shoot侧过渡带，紫色为颖果估计，红色为旧seed/base/root掩膜；黑色箭头表示自动估计的茎叶方向；
3. 青色圆点为保留的旧学习点，红叉为将排除的点，黑框为旧解码基部，黄色星号为符合新规则的旧学习起算点。

人工需要判断：地上部是否完整、颖果根须是否充分排除、黄色过渡区是否位于真正的芽鞘/茎基部shoot侧。页面使用独立浏览器本地存储并可导出CSV，不覆盖此前的路径审计记录。

## 6. 下一门槛

在用户完成或确认本页视觉审计前，不修改 G1′教师、不训练模型。若有效域通过，则按锁定顺序执行：

1. 将 `phenotype_roi` 约束接入train自动教师候选和跨增强共识；
2. 用同一核心配置重新生成train教师并在 RTX 3090 重新训练；
3. 仅在val比较根/颖果点污染、学习基部可用率、路径语义、漏叶、错连、重复性和重建误差；
4. val锁定前不读取test，不启动五随机种子和完整消融。
