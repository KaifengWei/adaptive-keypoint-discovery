# 基于DINOv2的秧苗自适应关键点发现完整实验方案

版本：1.1（表型驱动口径修订版）  
日期：2026-07-16  
状态：G1′候选阶段通过；进入点集到表型验证桥梁；未进入深度训练  

## 一、研究问题与路线重构

本研究只回答一个问题：

> 给定一幅秧苗图像，算法能否在没有人工关键点定义的情况下，根据当前植株的结构和形态，自主确定关键点的位置、数量及其稳定性？

这里的“关键点”不是人工命名的固定语义类别，而是为曲线重建和表型计算服务、由算法发现的二维支撑点集合。有效发现需要满足：

1. **实例自适应**：不同植株允许输出不同数量和不同位置的点；
2. **变换稳定**：同一植株经过旋转、缩放、平移和光照变化后，点能够对应回来；
3. **结构相关**：点主要分布在植株结构上，而不是背景、压缩噪声或偶然纹理上；
4. **路径可用**：点经骨架图分支和排序后，能够用于折线或样条曲线重建；
5. **表型充分**：用这些点计算长度、角度、曲率等表型时具有足够精度和稳定性。

本研究不要求点必须少量或具有逐点解剖学名称。点数由表型误差—复杂度关系决定：只要增加点仍能改善曲线或表型精度，较密的点就是合理支撑；只有增点不再带来收益或引入噪声、错误连线时，才称为冗余。

### 路线重构的自省结论

此前路线存在四个方法论问题：一是从已有几何代码反推研究方法，形成实现偏差；二是在核心假设未验证前堆叠多个大型组件；三是给尚未验证的技术组合过早命名，使概念原型看起来像成熟模型；四是把DINOv2当作后置增强，没有先系统检查自监督表征与无监督地标发现文献。

本版不沿用旧架构。调研报告的积极价值在于，它正确识别了DINOv2在稠密视觉表征、语义对应和部件发现方面的潜力；本版在此基础上补上动态数量机制、证据边界、可行性门禁和失败条件，而不是把调研报告当作需要对抗的对象。

## 二、证据基础与最终选型

### 2.1 已有研究真正提供了什么

- DINOv2提供无需下游标注的通用视觉特征，能够输出patch级稠密表示；官方模型卡建议把完整微调放在最后，优先使用冻结特征。ViT-S/14只有约21M参数，适合作为第一版编码器。  
  来源：[DINOv2论文](https://arxiv.org/abs/2304.07193)、[官方模型卡](https://github.com/facebookresearch/dinov2/blob/main/MODEL_CARD.md)
- 带register tokens的DINOv2能够减轻背景区域高范数伪影，使稠密特征图和注意力图更平滑，因此第一版优先使用`dinov2_vits14_reg`。  
  来源：[Vision Transformers Need Registers](https://arxiv.org/abs/2309.16588)
- PDiscoFormer证明DINOv2适合无监督部件发现，但它发现的是部件响应，不直接解决每幅图输出多少关键点。  
  来源：[PDiscoFormer](https://arxiv.org/abs/2407.04538)
- StableKeypoints与D-ULD++说明局部紧凑、变换等变、自训练对应等约束可以产生无监督地标，但多数现有方法仍以预设输出容量运行。  
  来源：[StableKeypoints](https://openaccess.thecvf.com/content/CVPR2024/html/Hedlin_Unsupervised_Keypoints_from_Pretrained_Diffusion_Models_CVPR_2024_paper.html)、[D-ULD++](https://openaccess.thecvf.com/content/CVPR2024/html/Tourani_Pose-Guided_Self-Training_with_Two-Stage_Clustering_for_Unsupervised_Landmark_Discovery_CVPR_2024_paper.html)
- DINOSAUR证明可以冻结自监督ViT并通过重建其特征学习object-centric slots；Adaptive Slot Attention进一步证明slot数量可以依据实例复杂度动态变化。  
  来源：[DINOSAUR](https://openreview.net/forum?id=b9tUk-f_aG)、[Adaptive Slot Attention](https://openaccess.thecvf.com/content/CVPR2024/html/Fan_Adaptive_Slot_Attention_Object_Discovery_with_Dynamic_Slot_Number_CVPR_2024_paper.html)

### 2.2 三条候选路线比较

| 候选路线 | 成熟度 | 能否动态决定数量 | 主要问题 | 本研究角色 |
|---|---:|---:|---|---|
| DINOv2注意力峰+非极大值抑制 | 高可实现、低方法成熟 | 阈值可产生不同数量 | 注意力显著性不等于结构关键点 | 可行性探针与强基线 |
| DINOv2特征聚类/无监督部件发现 | 中等 | HDBSCAN等方法可由密度决定簇数 | 容易得到区域中心而非紧凑地标 | 第二基线 |
| DINOv2+动态地标槽位 | 研究型、证据链较完整 | 可以 | 可能发生槽位塌缩或只分割器官 | 主方法候选 |

### 2.3 推荐主路线

第一版不命名新模型，采用以下中性描述：

> **冻结DINOv2编码器 + 动态高斯地标槽位 + 自监督稳定性与特征重建约束。**

处理流程为：

```text
秧苗RGB图像
  → 自动粗裁剪，保留完整植株
  → 冻结DINOv2 ViT-S/14-Reg提取稠密patch特征
  → 动态槽位模块从候选槽位中激活必要数量
  → 每个激活槽位解码为一个紧凑高斯热图
  → soft-argmax得到二维坐标和置信度
  → 输出本图专属的可变数量关键点集合
```

DINOv2是完整、独立的视觉编码器；动态槽位和坐标解码器是本研究需要训练的部分。注意力图只用于诊断和基线，主方法使用patch特征。

## 三、数据与人工工作

### 3.1 现有数据判断

当前已选核心目录包含400张原始秧苗图，主要分辨率为1024×768，但也包含纵向图、白底抠图、深色背景、裁切不完整和疑似掩膜残留等多种视觉域。现有生长期字段中有304张仍标记为待复核，因此不能直接按现有阶段标签进行正式分层。

新实验只读取原始RGB图像和新建清单，不读取旧关键点坐标、旧伪标签或旧训练角色。

### 3.2 重新建清单与划分

对400张图逐张完成数据审计，并按原始植株/近重复组拆分：

| 集合 | 数量 | 用途 |
|---|---:|---|
| 开发集 | 280 | DINOv2可行性探索与动态解码器训练；其中80张组成先导子集 |
| 验证集 | 60 | 冻结层选择、阈值、损失权重和停止轮次 |
| 锁定测试集 | 60 | 所有配置冻结后只运行一次 |

划分前必须用文件哈希、感知哈希和来源组排除近重复泄漏。背景类型、方向、植株占比、遮挡、裁切和形态复杂度在三组中保持基本平衡。视觉质量明显不合格的图不删除原文件，只在清单中标记`exclude_reason`。

### 3.3 人工需要做什么

| 工作 | 数量 | 实际操作 | 不需要做的事 |
|---|---:|---|---|
| 图像审计 | 400张 | 判断完整/截断、背景类型、是否单株、是否明显伪影 | 不点任何坐标 |
| 先导结果复核 | 80张 | 查看DINOv2响应图和候选点，记录落在植株/背景、重复点、明显失稳 | 不规定正确点应该在哪里 |
| 测试盲审 | 60张×3种方法×2人 | 随机查看完整方法和两个最强基线，共180个结果面板/人，按结构相关性、冗余和明显遗漏进行0/1/2评分 | 不参与调参 |
| 轮廓复核 | 最多60张 | 自动前景轮廓失败时修正二值轮廓，用于独立评价 | 不把轮廓输入主模型训练 |

因此，人工工作是“审计和评价算法发现了什么”，不是“替算法定义应该发现什么”。

## 四、分阶段实验与硬门禁

### G0：数据审计与预处理冻结

目标：建立与旧实验完全独立的新清单。

操作：

1. 统一EXIF方向，不强制旋转植株到同一方向；
2. 用颜色/背景差异获得粗前景框，只用于裁剪，四周保留15%边距；
3. 记录原图到模型输入的可逆坐标变换；
4. 生成400张审计接触表和数据分布统计；
5. 冻结划分文件及SHA-256。

门禁：可用单株图不足300张，或近重复无法可靠分组时，停止并补数据，不进入模型实验。

### G1：DINOv2训练前可行性试验

目标：先回答DINOv2在这些秧苗图像上是否真的提供稳定的局部结构线索。

数据：开发集中的80张先导图；其中20张只用于确定评价半径和显示参数，剩余60张用于先导结论。

模型与输入：

- `dinov2_vits14_reg`，参数完全冻结；
- 比较518×518和728×728两档输入；
- 比较最后一层patch特征与最后四层平均特征；
- 注意力图只作为诊断，不作为默认特征；
- 对每张图生成原图、水平翻转、±10°旋转、0.9/1.1缩放、轻微平移、亮度/对比度变化。

三个训练前基线：

1. CLS-to-patch注意力峰；
2. patch特征局部差异峰；
3. 在单幅图内部对patch特征和弱空间坐标做HDBSCAN，再取各簇medoid。聚类不跨图像直接混合二维坐标。

评价：把增强图候选点逆变换回原图，用匈牙利匹配计算重复率、定位漂移、数量差和植株区域命中率。

进入G2必须同时满足：

- 至少80%的先导图没有出现“全部落背景”或“全部聚成一个点”的失败；
- 最佳DINOv2特征候选的重复率F1相对注意力峰基线提高至少0.10；
- 几何增强后的中位定位误差不超过植株包围框对角线的5%；
- 光度增强前后的点数中位绝对差不超过1；
- 两名复核者认为至少70%的图存在可重复的结构相关候选区域。

阈值在20张校准图上冻结，不能根据其余60张结果反复修改。任一关键门禁失败，则DINOv2只保留为负结果或普通特征基线，不直接进入复杂训练。

### G1.5：G1′点集到表型验证桥梁

目标：判断当前自动候选点是否能够形成有序分支路径，并准确计算目标表型。

1. 导出每个点的坐标、类型、得分及对应前景和骨架；
2. 将点投影到最近骨架像素，构建8邻域骨架图；
3. 识别端点、分叉点和路径，将普通支撑点分配到路径并按测地距离排序；
4. 分别使用折线、B样条和平滑样条重建路径；
5. 计算路径长度、切线角、曲率、弧弦比，并结合掩膜计算宽度和面积；
6. 先与全像素骨架参考比较，再与独立人工表型测量比较。

G1.5门禁不是“点是否足够少”，而是样条曲线和表型误差是否达到要求、扰动前后表型是否稳定。当前20张干净背景核心集保持不变；先在6张smoke完成接口，再一次性运行剩余14张。

### G2：动态高斯地标槽位原型（仅在G1′表型效用不足时启用）

目标：使模型在无关键点标签条件下，自主决定每幅图激活多少个地标槽位。

架构：

1. 冻结DINOv2，为每张训练图预计算带可逆变换矩阵的增强视图特征库；训练动态解码器时不重复加载完整编码器，但几何一致性仍来自DINOv2真实处理过的增强图，而不是简单扭曲同一张特征图；
2. 设定32个候选槽位作为计算容量上限，不代表输出必须有32个点；验证集另做16/32/48容量敏感性检查，证明结论不由上限人为决定；
3. 使用Adaptive Slot Attention式离散采样决定激活槽位；
4. 单独设置背景槽位，避免有效槽位浪费在背景；
5. 每个激活槽位输出一个二维高斯热图，中心经soft-argmax得到坐标；
6. 用激活槽位和热图重建DINOv2 patch特征。

总损失只保留六项：

\[
L=\lambda_{rec}L_{feature}+\lambda_{eq}L_{equiv}+\lambda_{cmp}L_{compact}+\lambda_{sep}L_{separate}+\lambda_{card}L_{cardinality}+\lambda_{bg}L_{background}.
\]

| 损失 | 作用 |
|---|---|
| `L_feature` | 用地标槽位重建冻结DINOv2特征，保证发现结果包含视觉结构信息 |
| `L_equiv` | 同一图两个增强视图经匈牙利匹配后坐标一致 |
| `L_compact` | 每个槽位热图集中在局部，防止退化成整片区域 |
| `L_separate` | 抑制多个槽位占据同一位置 |
| `L_cardinality` | 以曲线和表型收益约束新增槽位；不追求绝对少点，只抑制不改善任务结果的增点 |
| `L_background` | 背景patch由背景槽位解释，减少背景假点 |

训练顺序：前10个epoch只训练特征重建、紧凑和分离；随后逐步加入等变与数量代价，避免一开始全部关闭或全部激活。DINOv2全程冻结。先运行3个种子的20张过拟合试验，通过后再训练开发集；正式比较使用5个种子。

G2门禁：

- 3个种子均无全部槽位关闭、全部槽位开启或所有点重合；
- 光度变化不改变植株形态时，点数中位绝对差不超过1；
- 旋转/缩放重复率不低于G1最佳训练前候选；
- 输出点数与图像像素尺寸、背景亮度的相关性接近0；
- 输出点数随独立形态复杂度指标变化，而不是只随植株在画面中的面积变化。

### G3：比较、消融与失败分析

正式比较方法：

1. SIFT/ORB兴趣点（传统下界）；
2. DINOv2注意力峰；
3. DINOv2局部特征峰；
4. DINOv2单图HDBSCAN特征簇medoid；
5. StableKeypoints公开实现，若能够按原论文环境可靠复现；
6. 完整动态地标槽位方法。

核心消融只做五项：

1. DINOv2-Reg改为普通DINOv2；
2. 去特征重建；
3. 去等变一致性；
4. 去动态槽位采样，保留全部候选槽位；
5. 去紧凑/分离约束。

消融与表征比较分开报告，不把“换编码器”和“删除方法机制”混为一类。

### G4：锁定测试

验证集确定模型层、输入分辨率、阈值、损失权重和停止轮次后，生成配置哈希。测试集只运行一次，不允许根据测试结果修改模型。保存所有失败图，不能只展示成功案例。

## 五、指标、统计与成功判据

### 5.1 主指标

| 指标 | 定义 |
|---|---|
| 变换重复率F1 | 两个视图点集逆变换后，在归一化距离阈值内的匹配精确率与召回率 |
| 归一化定位误差 | 匹配点距离除以植株包围框对角线 |
| 点数稳定性 | 不改变形态的增强前后，激活点数的绝对差 |
| 结构重建误差 | 从独立前景轮廓提取骨架，以前景内测地最小生成树连接点集，计算其与密集骨架的Chamfer与Hausdorff误差 |
| 点数—结构Pareto面积 | 同时考察更少点数与更低结构重建误差，不单独追求点少或点多 |

### 5.2 自适应性证明

只证明“点数变化”还不够。需要排除以下替代解释：图像更大、背景更亮、裁剪更松或噪声更多导致点数增加。

独立计算五项形态复杂度：前景骨架总长度、分支/端点数、曲率熵、轮廓复杂度和植株占框比例。使用多元回归或混合效应模型检验：在控制占框比例、背景和图像尺寸后，输出点数是否仍与结构复杂度显著相关。

### 5.3 专家盲审

两名复核者只看到随机打乱的方法结果，不知道是哪一种方法。每幅图按以下三项0/1/2评分：结构相关性、明显冗余、明显遗漏。报告加权Cohen's kappa和分歧案例，不以专家意见替代量化指标。

### 5.4 统计

- 正式结果使用5个种子；
- 以植株为单位进行1000次bootstrap，报告95%置信区间；
- 多方法比较使用配对检验和Holm校正；
- 主结论要求完整方法在“重复率—定位误差—点数—结构误差”Pareto前沿上优于训练前DINOv2候选，且95%置信区间不跨越预登记非劣界；
- 不用单一综合分数掩盖某项明显失败。

## 六、算力、产物与执行节奏

### 6.1 算力

推荐从DINOv2 ViT-S/14-Reg开始。官方模型约21M参数、patch size 14。G1只做冻结推理，8GB级CUDA设备通常可以从小batch开始测试；G2将DINOv2特征预计算到磁盘，只训练小型动态解码器，建议12—16GB显存、32GB内存和100GB可用磁盘。728分辨率是否可用必须由显存预检决定，不在论文中凭估算宣称。

| 第6周 | 锁定测试、统计、失败分析和论文图表 | G4 |

## 七、风险与停止规则

| 风险 | 识别信号 | 处理或停止规则 |
|---|---|---|
| 叶片太细，patch分辨率不足 | 点总落在宽区域中心，叶尖漂移大 | 先比较728输入与多尺度裁剪；仍失败则DINOv2不适合作为定位主干 |
| 背景伪影主导 | 点数与背景亮度强相关 | 使用register模型和背景槽位；验证集仍相关则停止 |
| 槽位塌缩 | 全关、全开或全部重合 | 分阶段加损失；3个种子仍塌缩则放弃动态槽位路线 |
| 输出只是器官区域中心 | 热图稳定但不紧凑、结构重建差 | 加强高斯瓶颈；仍失败则将结果降级为部件发现而非关键点发现 |
| 数量变化没有结构依据 | 点数只随面积/分辨率变化 | 不能声称实现自适应发现 |
| 专家认为点有意义但量化不稳定 | 盲审高、重复率低 | 不能以主观评价替代稳定性证据 |
| 点集稳定但不能形成正确路径 | 重复率高、分支连接或排序错误 | 只能称为候选点，不能声称已支撑表型提取 |
| 点集较密 | 点数较多但覆盖和重复性良好 | 不直接判失败；检查表型误差是否随点数下降并在何处饱和 |

最终原则：先证明候选点在真实秧苗图上稳定覆盖有效结构，再证明点集能够形成正确路径并改善表型计算。点数多少不是独立目标，最终依据是自动表型的精度、稳定性及其对秧苗评价的贡献。只有G1′无法满足这些任务要求时，才进入更复杂的动态数量训练模型。
