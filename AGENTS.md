# 仓库级 Codex 工作约束

本仓库只服务于秧苗自适应关键点发现小论文。任何账号、模型、设备或 Codex 任务在提出方案、修改代码或启动实验前，必须先完整阅读：

1. `adaptive_keypoint_discovery_reboot/AGENTS.md`；
2. `adaptive_keypoint_discovery_reboot/ACCOUNT_HANDOFF.md`；
3. `adaptive_keypoint_discovery_reboot/PROJECT_STATE.md`；
4. 上述交接文件列出的训练报告和核心结果文件。

## 不得静默改变的研究事实

- 唯一研究问题是：在没有人工关键点定义和坐标标签的情况下，从单株秧苗图像中自适应发现数量可变、位置稳定、能够支撑表型计算的关键点。
- DINOv2 是视觉骨干/表征来源；G1′是自动教师和结构候选基线；学习型动态热图检测器负责学习教师目标。不得混淆三者角色。
- V3 数据集、自动教师目标和 RTX 3090 首轮核心训练均已完成；当前瓶颈是部分叶片路径连接错误，表型准确率仍待人工参考验证。
- `test` 不得用于调参；跨作物迁移已封存；不得恢复预定义固定关键点路线。
- 研究问题、方法架构、教师目标、数据划分、评价指标或论文主结论的改变，必须先给出证据和备选方案，并得到用户确认。
- 有效结果、失败结论和路线变化必须写入 `PROJECT_STATE.md` 并提交 Git，不能只保留在聊天记忆中。

详细执行规则以子目录 `AGENTS.md` 为准；发生冲突时，以用户当前明确指令为最高优先级。
